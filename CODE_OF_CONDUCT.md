# Code of Conduct

OpenVoiceFlow follows the [**Contributor Covenant**](https://www.contributor-covenant.org/), version 2.1, which you can read in full at <https://www.contributor-covenant.org/version/2/1/code_of_conduct/>.

## In short

Be respectful. Assume good faith. Help newcomers when you can. Disagree about ideas, not people. Keep technical discussions technical.

If something feels off — a comment in a PR, an exchange in Discussions, or anything else — please report it through the channels below. Reports are handled confidentially.

## Scope

This applies in all project spaces — the GitHub repository, Issues, Pull Requests, Discussions, and any other space where someone is acting on behalf of the project.

## Reporting

For most matters, open a GitHub Discussion or contact the maintainer directly.

For anything sensitive — including conduct concerns where you don't want to be public — use GitHub's private security advisory channel (the same one described in [SECURITY.md](SECURITY.md)). The maintainer will respond within 5 business days.

> **For Mohit before publishing:** if you want to add a dedicated conduct-reports email separate from the security email, drop it in here. Without one, all reports flow through the security advisory channel.

## Enforcement

The maintainer follows the **Community Impact Guidelines** from the Contributor Covenant. Consequences range from a private warning to a temporary ban to a permanent ban, depending on severity and repetition. The full guidelines are in the linked Covenant document.

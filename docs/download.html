Source install for repo collaborators

The repository is private during this launch phase, so public users should use the website-hosted DMG downloads above. If you already have GitHub access, you can install from source.

git clone https://github.com/shimoverse/openvoiceflow.git && cd openvoiceflow && bash install.sh

Download notes and limitations

  • The website-hosted DMGs above are copied from the OpenVoiceFlow v0.3.0 build so public visitors can download them while the source repo stays private during this launch phase.
  • This page now links the v0.3.0 website-hosted DMGs built from the current tested main branch.
  • OpenVoiceFlow supports macOS 12+ on Apple Silicon and Intel. Windows and Linux downloads are not currently available.
  • The current v0.3.0 DMGs are unsigned and not notarized. The release pipeline now supports Developer ID signing and Apple notarization when the maintainer provides Apple Developer credentials.

Download FAQ

Is the source repository public?

Not yet. OpenVoiceFlow is intended to be MIT licensed, but the GitHub repository is private during this launch phase. Public visitors should use the website-hosted DMG downloads.

Does OpenVoiceFlow send audio to the cloud?

No. Audio transcription runs locally through whisper.cpp. Cleaned transcript text only goes to a cloud LLM if you choose a cloud backend.

Which backend should I choose?

The current default cloud cleanup path is OpenRouter Gemma 4. Ollama and the “none” backend keep cleanup local/no-cloud.

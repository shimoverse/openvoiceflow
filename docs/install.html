Install guide · macOS 12+

Install OpenVoiceFlow on macOS

Install OpenVoiceFlow by downloading the macOS DMG from this website. The app needs microphone access, whisper.cpp for local transcription, and either OpenRouter, another LLM provider, Ollama, or no cleanup backend. Source install is only for collaborators with private repo access.

Download for macOSDownload guide

Fast path: DMG install

  1. Open the download page and pick Apple Silicon or Intel.
  2. Download the DMG from this website.
  3. Open the DMG and drag OpenVoiceFlow into Applications.
  4. Launch OpenVoiceFlow and grant macOS microphone/accessibility permissions when prompted.
  5. Choose your cleanup backend: OpenRouter Gemma 4, Groq, OpenAI, Anthropic, Ollama, or none.

Terminal install from source

git clone https://github.com/shimoverse/openvoiceflow.git && cd openvoiceflow && bash install.sh

Source install currently requires access to the private GitHub repository. Public website visitors should use the DMG download path.

Manual setup

brew install whisper-cpp
git clone https://github.com/shimoverse/openvoiceflow.git && cd openvoiceflow
python3 -m venv ~/.openvoiceflow/venv
~/.openvoiceflow/venv/bin/pip install -e ".[all]"
openvoiceflow --backend openrouter --set-key openrouter YOUR_KEY_HERE
openvoiceflow

Troubleshooting

  • If macOS says the app cannot be opened or should be moved to Trash, the hosted v0.3.0 DMG is not notarized yet. Right-click Open is the temporary workaround until the next Apple-notarized DMG is published.
  • If transcription does not start, confirm microphone permission in macOS System Settings.
  • If paste fails, grant Accessibility permission for the app or terminal launching OpenVoiceFlow.
  • If cloud cleanup fails, verify your chosen provider key; OpenRouter uses openrouter, not a Gemini API key.
  • If you need a fully local flow, choose Ollama or --backend none.

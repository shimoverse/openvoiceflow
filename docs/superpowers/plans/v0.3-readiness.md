# Plan — OpenVoiceFlow v0.3.0 Execution

> **Status:** Active. Branch: `v0.3-readiness`. Pairs with `docs/superpowers/specs/v0.3-readiness.md`.
> TDD discipline throughout: tests fail first, then pass. Lint blocking. Verification before completion.

## Wave order (locked)

1. **Wave 0** — Test scaffold + commit discovery layer
2. **Wave 1** — v0.1 ship-stoppers still open + v0.2 ship-stoppers introduced (SS2, SS3, SS5, SS6) + small Wave-1 fixes (D10, D12 opt-out)
3. **Wave 2** — Privacy-by-default toggles (chmod 600, default-off logs, consent-gated auto-learn, README hero rewrite, key validation, `openvoiceflow doctor`, lint cleanup)
4. **Wave 3** — Community-health files (parallel subagents, fresh context per file)
5. **Wave 4** — Test coverage push + CI gating + matrix
6. **Wave 5** — Release infrastructure (PyPI Trusted Publisher, brew tap, RELEASE.md, version-tag enforcement)
7. **Wave 6** — Pre-publish 4-agent parallel review pass; cut `v0.3.0-rc1` then `v0.3.0`

Each wave commits independently. No `git push` without Mohit's explicit ask.

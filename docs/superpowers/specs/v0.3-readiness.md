# Spec — OpenVoiceFlow v0.3.0 "Ready for the World"

> **Status:** Active. Branch: `v0.3-readiness`. Author: Claude (working with Mohit).
> Created: 2026-04-27.
> Pairs with: `docs/superpowers/plans/v0.3-readiness.md` and `docs/superpowers/audits/{PROJECT_AUDIT,PERSONA_AUDIT,READINESS_CHECKLIST}.md`.

## What this is

The single bundle that takes OpenVoiceFlow from "v0.2 exists on a private GitHub" to "v0.3.0 is published, installable from PyPI / Homebrew / DMG, and survives a procurement review at a small company." Same bar Mohit held [snapfeed](https://github.com/mohitjain/snapfeed) to.

This spec is the **what** and **why**. The **how** lives in the plan.

## Goals

1. **Every install path advertised in the README actually works on a clean Mac.** No 3.9 import crashes, no broken `~/.local/bin/openvoiceflow` shim, no upgrade that silently drops a user's custom prompt.
2. **A privacy-conscious default state for new users.** Plaintext API keys at mode 600. Transcript logs default off. Auto-learner consent-gated. Hero claim in README accurately matches the runtime data flow.
3. **Procurement-reviewable.** SECURITY.md, PRIVACY.md, THREAT_MODEL.md, COMPLIANCE.md, CONTRIBUTING.md, CODE_OF_CONDUCT.md, CHANGELOG.md, VERSIONING.md, COMPATIBILITY.md, AGENTS.md, legal/{DPA-template.md, THIRD_PARTY_NOTICES.md} all exist and answer the questions a small-team CISO would ask.
4. **Tested.** Pytest scaffold, mocked-HTTP backend tests, config-migration round-trip test, transcriber argv assembly test, learner correction-extraction test, build/install smoke for Python 3.9, 3.10, 3.11. CI gates lint and tests.
5. **Releasable in one command.** `git tag v0.3.0 && git push --tags` ⇒ wheel + sdist on PyPI (Trusted Publisher OIDC, no token), DMGs attached to GitHub Release, brew tap formula updated. RELEASE.md documents the happy path and the "when things go wrong" table.
6. **AI-coding-agent friendly.** AGENTS.md gives Claude/Cursor/Copilot a clear file map and "to add a backend, edit these 4 files" recipe.

## Non-goals (explicit out-of-scope)

- DMG signing / Apple notarization (Decision D6 = defer). First-launch Gatekeeper override stays. We document it prominently.
- Switching to macOS Keychain for API-key storage (Decision D5 = defer). chmod 600 plaintext is good enough for v0.3.0.
- Real telemetry / phone-home (Decision D12 = formally none). Update check stays but becomes opt-out.
- Windows / Linux / iOS / Android. Still macOS-only.
- New features. v0.3.0 is the readiness release; the only "feature" that ships is opt-in/opt-out toggles for existing v0.2 features.
- Repo transfer / GitHub-owner change (Decision D1 = defer; one find-replace at transfer time).
- Changing the auto-learner algorithm or extending it.
- Adding a new LLM backend.

## Success criteria (objective, falsifiable)

A v0.3.0 release succeeds if and only if all of the following hold on the day of the tag:

| # | Criterion | How verified |

| S6 | `ruff check voiceflow/` returns 0 errors (lint becomes blocking) | CI |
| S7 | A user with a v0.2.0 `~/.openvoiceflow/config.json` containing `cleanup_prompt` upgrades to v0.3.0 and their custom prompt is preserved as `llm_prompt` | unit test in `tests/test_config_migration.py` |
| S8 | A fresh-install user has `log_transcripts: false` and `auto_learn: false` until they explicitly opt in via the onboarding wizard | unit test + manual smoke |

## Decisions (locked)

D1 = defer (keep `shimoverse/openvoiceflow` URLs).
D2 = reserve PyPI `openvoiceflow` ahead of release.
D3 = stay on `requires-python = ">=3.9"`; add `from __future__ import annotations` to all 15 files using `X | None`.
D4 = keep Gemini default backend; rewrite README hero to clarify the data flow.
D5 = chmod 600 plaintext; no `keyring` dep this round.
D6 = no DMG signing.
D7 = full v0.3.0 bundle (Waves 1–6).
D8 = `log_transcripts` default false for fresh installs; existing users keep their setting.
D9 = `auto_learn` default false; consent-gated in the Know Me interview.
D10 = count actual default voice commands; reconcile README + landing page + commands.py.
D11 = keep current Anthropic Haiku + Groq Llama-3.1-8B defaults.
D12 = formally commit "no telemetry" in PRIVACY.md; document update-check; add `update_check: false` opt-out.

## Constraints

- **TDD for every code change.** Tests fail first, then pass. Lint is blocking. Verification before completion.
- **No `git push` without explicit ask** (Mohit's standing instruction). Commit freely on `v0.3-readiness`. The branch stays local until Mohit asks to publish.
- **Plain-English status updates** between waves. Mohit is a product thinker, not a stats engineer.
- **No new feature work.** Toggles + defaults + docs only.
- **Subagent-driven development for parallel work** — per-doc agents in Wave 3, per-domain agents in Wave 6.
- **Two-stage review per task** (per Mohit's standing instruction): spec compliance + code quality.

## Alternatives considered

- **Cut v0.2.1 with only ship-stoppers, then v0.2.2 / v0.2.3 with the rest.** Rejected — the ship-stoppers + missing community-health files + missing tests are the same blockers; dribbling them out reads as ongoing patchwork rather than a "we're public now" moment. (Decision D7.)
- **Bump min-Python to 3.10 and remove `from __future__`.** Rejected — macOS 12 ships Python 3.9 by default; bumping to 3.10 forces every install path to first install a newer Python. Future-annotations is one trivial line per file. (Decision D3.)
- **Move keys to macOS Keychain via `keyring`.** Rejected for v0.3.0 — adds a runtime dep, complicates the DMG bootstrap, and chmod 600 already closes the world-readable footgun. Revisit for v0.4. (Decision D5.)
- **Default off both `log_transcripts` AND `auto_learn` AND `update_check`.** Adopted for first two; `update_check` stays default-on but opt-out (a single anonymous public-API request to GitHub Releases is industry-standard for auto-update and is already silent on failure).
- **Keep telemetry-free as a pitch but ship a "did install succeed" anonymous beacon.** Rejected — Mohit's audience cares about the principle, not just the practice. (Decision D12.)

## Risks and mitigations

| Risk | Mitigation |
|---|---|
| `from __future__ import annotations` breaks runtime annotation introspection | Verified: no Pydantic / FastAPI / attrs / inspect.signature usage in the codebase |
| chmod 600 breaks if a user's home dir has unusual ACLs | Wrap in try/except; log a warning on failure; do not crash |
| Default-off log_transcripts surprises v0.2 users on upgrade | Migration logic respects existing config; defaults apply only to fresh installs |
| Auto-learner consent screen lengthens the Know Me flow | Single screen, two buttons ("Yes, learn from my edits" / "No, just dictate"); ~5 seconds added |
| PyPI Trusted Publisher needs the project to exist first | Reserve `openvoiceflow` on PyPI manually before configuring OIDC |
| Test matrix on 3.9 is slow | All tests are unit/mocked; ~30 s total even on 3.9 |
| Subagent-generated docs are generic boilerplate | Each agent prompt includes the audit findings; reviews check for OpenVoiceFlow-specific content |
| `pip install` from PyPI on day 1 must work | Wave 5 ends with a dry-run release tag (`v0.3.0-rc1`) that exercises the full pipeline before the real `v0.3.0` |
